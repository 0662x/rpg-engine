# Authoring Boundaries

本文定义 `isekai-farm` 原生剧情包与存档包的维护边界。

## 剧情包负责

- 世界、规则、实体模板、路线、随机表、GM prompt、行动模板和 smoke tests。
- 可以被多个存档复用的稳定内容。
- 新开局时应被初始化进存档的基础状态。

## 存档包负责

- 某一局的当前状态、事件历史、数据库、快照和卡片。
- 当前天数、当前位置、库存数量、进度钟数值、人物关系数值和已发生事件。
- 当前角色关系、承诺、同居/关押/驯化进展、当前陷阱状态、当前植物位置和当天优先事项。
- 由运行时生成的 `snapshots/`、`cards/` 和 `data/`。

## 不要混放

- 剧情包内不要加入 `data/`、`cards/`、`snapshots/`、`memory/`、`reports/`、`backups/` 或 `save.yaml`。
- 剧情包内不要写 `rel:pc-*` 当前关系记录；通用社交设计只能放在规则、随机表、palette 或 reference 中。
- 存档包内不要直接编辑作者内容源；存档中的 `campaign.yaml` 是运行用视图。
- 不要为了整理而改写历史事件日志。历史 payload 可以保留当时的来源描述。

## 当前协作约束

游戏内容补全与设定修订需要和玩家一起完成。没有明确来源或玩家确认前：

- 不新增世界事实。
- 不重命名实体 ID。
- 不改写 starter 开局内容。
- 不调整当前存档事实。

允许先做工程维护：文档、质量门禁、目录卫生检查、导出导入检查、发布流程说明。
